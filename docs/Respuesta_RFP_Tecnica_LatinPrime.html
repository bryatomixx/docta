2.5 Automatización, IA y enriquecimiento de datos

n8n / Make / Zapier / GHL workflows:
✓ n8n (self-hosted, mayor control, transferible).
Operaciones mensuales estimadas para 150 leads/día:
✓ ~4,500 leads/mes → ~20,000–30,000 operaciones/mes (enriquecimiento + IA + mensajería + seguimientos).
Automatizaciones incluidas en Fase 1:
✓ Entry del formulario de Facebook (Lead Ads) → enriquecimiento → cálculo del estimado → redacción IA → envío del estimado (SMS+email) → seguimiento → opt-out → avance de pipeline.
Qué agentes de IA proponen:
✓ Un agente de calificación/oferta (texto). Voz (VAPI) en fase posterior.
¿La IA solo responde, o también califica / pide datos / interpreta / sugiere oferta / prioriza?
✓ Todo lo anterior: califica, pide datos faltantes, interpreta respuestas, sugiere rango de oferta y prioriza (ej. dueño ausente + alto equity).
Modelo de IA (OpenAI / Claude / Gemini):
✓ Claude (Anthropic) — Opus 4.8 para ofertas, Haiku para SMS.
Cómo evitan respuestas incorrectas / promesas indebidas / errores legales:
✓ El monto de la oferta se calcula en código (determinista); la IA NO inventa cifras; lenguaje obligatorio "preliminar / no vinculante / sujeto a inspección"; guardrails en el prompt.
¿Revisión humana antes de una oferta formal?
✓ Regla dura: la IA NUNCA cierra ni envía la oferta final — SIEMPRE la da un humano. La IA solo manda el estimado preliminar inicial y (mes 2) califica/responde. Cuando un lead muestra interés, un sistema de notificaciones alerta al humano (dashboard + email/Slack/WhatsApp) para que entre, negocie y envíe la oferta final.
Fuentes de enriquecimiento (Zillow, BatchLeads, PropStream, Foreclosure.com, county, APIs, scraping):
✓ BatchData/BatchLeads (skip tracing + propiedad) y ATTOM (hipoteca, equity, propietario, county records). PropStream/Foreclosure.com integrables si Docta los prefiere. Zillow/MLS no viable como API abierta (gated); no dependemos de scraping.
Qué se automatiza vs. revisión manual:
✓ Automático: enriquecimiento, valuación, estimado preliminar, mensajería y calificación. Manual (siempre humano): la oferta final y el cierre.
Ficha automática por propiedad:
✓ Sí (funcional hoy).
